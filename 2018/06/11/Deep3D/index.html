Deep3D Fully Automatic 2D-to-3D Video Conversion CVPR2016

2018-06-11

  1. 1. Model Architecture
  2. 2. Selection Layer

Deep3D is trained directly on stereo pairs from a dataset of 3D movies to minimize the pixel-wise reconstruction error of the right view when given the left view.

We should first notice that inferring depth (or disparity) from a single image is a highly under-constrained problem. In addition to depth ambiguities, some pixels in the novel view correspond to geometry that’s not visible in the available view, which causes missing data that must be hallucinated with an in-painting algorithm.

In spite of these difficulties, our intuition is that given the vast number of stereo-frame pairs that exist in already-produced 3D movies it should be possible to train a machine learning model to predict the novel view from the given view. To that end, we design a deep neural network that takes as input the left eye’s view, internally estimates a soft (probabilistic) disparity map, and then renders a novel image for the right eye.

Model Architecture

Our network has a branch after each pooling layer that unsamples the incoming feature maps using so-called “deconvolution” 反卷积 layers (i.e., a learned unsampling filter). The upsampled feature maps from each level are summed together to give a feature representation that has the same size as the input image.

We perform one more convolution on the summed feature representation and apply a softmax transform. The output of this softmax layer is interpreted as a probabilistic disparity map. And then feed this disparity map and the left view to the selection layer, which outputs the right view.
