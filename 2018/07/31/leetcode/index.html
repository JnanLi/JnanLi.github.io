2 两数相加

给定两个非空链表来表示两个非负整数。位数按照逆序方式存储，它们的每个节点只存储单个数字。将两数相加返回一个新的链表。

你可以假设除了数字 0 之外，这两个数字都不会以零开头。

示例：

输入：(2 -> 4 -> 3) + (5 -> 6 -> 4)
输出：7 -> 0 -> 8
原因：342 + 465 = 807

4. 两个排序数组的中位数

给定两个大小为 m 和 n 的有序数组 nums1 和 nums2 。
请找出这两个有序数组的中位数。要求算法的时间复杂度为 O(log (m+n)) 。
你可以假设 nums1 和 nums2 均不为空。

示例 1:
nums1 = [1, 3]
nums2 = [2]
中位数是 2.0

示例 2:
nums1 = [1, 2]
nums2 = [3, 4]
中位数是 (2 + 3)/2 = 2.5

  • 注意vector的初始化方式，用string或int数组去初始化的时候，给出首尾地址
  • 将两个vector合并，可以用insert的方式
  • median = 5/2，这里这样是错误的，应该是mdian = 5/2.0。

时间复杂度

时间频度：这里都是定性的评估，一个算法耗费的时间从理论上是不能算出来的，我们的出发点是评估哪个算法耗费时间多，哪个算法耗费时间少。一个算法花费的时间与算法中语句的执行次数成正比例，一个算法中语句执行次数称为时间频度，记为T(n)。

时间复杂度：前面提到的T(n)随着n(问题的规模)变大会跟着一起变大，有时我们想知道它变化呈什么规律，为此我们引入时间复杂度的概念。若有某个辅助函数f(n)，使得当n趋近于无穷大时，T(n)/f(n)的极限值为不等于0的常数，我们称f(n)是T(n)的同数量级函数，记做T(n) = O(f(n))。它称为算法的渐进时间复杂度，简称时间复杂度。

算法复杂度可以从最理想情况、平均情况、最坏情况三个角度去评估，由于平均情况大多和最坏情况持平，而且评估最坏也可以避免后顾之忧，因此一般情况下，我们直接评估最坏情况。

大O表示法O(f(n))中的f(n)的值可以为1、n、log(n)、n2等，因此我们将之分为称为常数阶、线性阶、对数阶和平方阶。

推导大O阶，我们按照如下的规则进行推导：

  1. 用常数1来取代运行时间中所有加法常数
  2. 修改后的运行次数函数中，只保留最高阶项
  3. 如果最高阶项存在且不是1， 则去除与这个项相乘的常数

n+(n-1)+(n-2)+(n-3)+……+1
=(n+1)+[(n-1)+2]+[(n-2)+3]+[(n-3)+4]+……
=(n+1)+(n+1)+(n+1)+(n+1)+……
=(n+1)n/2
=n(n+1)/2
=n²/2+n/2
根据规则2，只保留最高阶n²/2；根据规则3，去掉常数1/2，最终这段代码的复杂度是O(n²)。

其他常见复杂度
除了常数阶、线性阶、平方阶、对数阶，还有如下时间复杂度：
f(n)=nlogn时，时间复杂度为O(nlogn)，可以称为nlogn阶。
f(n)=n³时，时间复杂度为O(n³)，可以称为立方阶。
f(n)=2ⁿ时，时间复杂度为O(2ⁿ)，可以称为指数阶。
f(n)=n!时，时间复杂度为O(n!)，可以称为阶乘阶。
f(n)=(√n)时，时间复杂度为O(√n)，可以称为平方根阶。

O(n)、O(logn)、O(√n)、O(nlogn)随着n的增加，复杂度提升不大，因此这些复杂度属于效率高的算法，反观O(2ⁿ)和O(n!)当n增加到50时，复杂度就突破十位数了，这种效率极差的复杂度最好不要出现在程序中，因此在动手编程时要评估所写算法的最坏情况的复杂度。
