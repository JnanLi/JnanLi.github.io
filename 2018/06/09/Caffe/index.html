Caffe简介

Blob Layer Net Solver
存储数据及其衍生物 传输bottom blobs到top blobs 由许多Layers组成，计算正向传播和反向传播的梯度 更新权重

Caffe Model Zoo

AlexNet VGG GoogleNet ResNet …

数据层详解

卷积层

输入： n×c0×w0×h0
输出： n×c1×w1×h1
其中，输出c1就是num_output。
w1 = (w0 + 2×pad - kernel_size) / stride + 1
h1 = (h0 + 2×pad - kernel_size) / stride + 1

reshape层

有一个可选的参数组shape，用于指定blob数据的各维度的值（blob是一个四维的数据：32×3×28×28）。
dim: 0 表示维度不变，即输入和输出是相同的维度
dim： -1 表示由系统自动计算维度，数据的总量不变，系统会根据blob数据的其它三维来自动计算当前维的维度值。
输出数据为： 32×3×14×56

Solver

Caffe提供了六种优化算法来求解最优参数，在solver配置文件中，通过设置type类型来选择。

Algorithm type
Stochastic Gradient Descent SGD
AdaDelta AdaDelta
Adaptive Gradient AdaGrad
Adam Adam
RMSprop RMSprop

网络配置文件位置

net: “examples/mnist/lenet_train_test.prototxt”
也可以分别设置为train_net和test_net：
train_net: “examples/mnist/lenet_auto_train.prototxt”
test_net: “examples/mnist/lenet_auto_test.prototxt”

迭代多少个测试样本

test_iter: 100
假设有5000个测试样本，一次测试想跑这5000个则需要设置test_iter×batch=5000

测试间隔

test_interval: 1000
也就是每训练1000次，才进行一次测试。

基础学习率

base_lr： 0.01

学习率调整的策略

lr_policy： “inv”

policy 说明
fixed 保持base_lr不变
step 还需要设置stepsize，base_lr * gamma ^ (floor(iter / stepsize))
exp base_lr * gamma ^ iter
inv 还需要设置power，base_lr (1 + gamma iter) ^ (-power) 常用
multistep 还需要设置stepvalue。
poly 略
sigmoid 略

动量

momentum： 0.9

每训练多少次，显示一下

display： 500

最大迭代次数

max_iter: 100000

快照

snapshot： 1000
将训练出来的model和solver状态进行保存
snapshot_prefix: “examples/mnist/lenet”

运行模式

solver_mode: CPU
