2. 面向过程的编程风格

主要介绍独立函数，独立函数的好处在于1.取代重复编写相同的程序代码，可使程序更易懂；2.可以在不同的程序中使用；3.更容易分工协作。

2.1 如何编写函数

当函数返回值类型是void的时候，return语句用来提前结束函数的执行。

如果函数的返回类型不为void，那么它必须在每个可能的退出点上将值返回。函数中的每条return语句都被用来明确表示该处就是函数的退出点。如果函数体的最后一条语句不是return，那么最后一条语句之后便是该函数的隐式退出点。

另外如果没有在main()的末尾写下return语句，这一语句会被自动加上。

2.2 调用函数

本节将实现一个可对vector内的整数值加以排序的函数，通过这个例子，我们可以审视两种参数传递方式：传址及传值。

当我们调用一个函数时，会在内存中建立起一块特殊区域，称为”程序堆栈”，这块特殊区域提供了每个函数参数的储存空间。它也提供了函数所定义的每个对象的内存空间——我们将这些对象称为local object。一旦函数完成，这块内存就会被释放掉，或者说是从程序堆栈中被pop出来。

当我们将参数列表中的对象传入函数，默认情形下其值会被复制一份，成为参数的局部性定义。这种方式称为传值。

只要在类型名称和reference之间插入&符号，就是声明了一个reference。C++不允许我们改变reference所代表的对象，它们必须从一而终。

如果以传值方式传入display()之中，这意味着每当我想进行显示操作时，vector内的所有元素都会被复制。这并不会造成错误，但是如果直接传入vector的地址，速度会更快。

pointer参数和reference参数之间更重要的差异是，pointer可能不指向某个实际对象，当我们提领pointer时，一定要先确定其值并非0。

当函数返回一个对象的时候，不论以pointer或reference形式将对象返回，都不正确。因为在函数执行完毕时已经不存在了。如果以传值方式返回，便不会产生问题：因为返回的是对象的副本，在函数之外依然存在。

2.2.1 作用域及范围

为对象分配的内存，其存活时间称为范围(extent)。

对象在程序内的存活区域称为该对象的作用域(scope)。

局部性范围（local extent）：每次函数执行，都会为对象分配内存，每当函数结束便会加以释放，我们称此对象具有局部性范围。函数参数便具有局部性范围。

函数参数也是具有局部作用域。

对象如果在函数以外声明，具有所谓的file scope，从声明点至文件末尾都是可见的。

file scope内的对象也具备所谓的static extent，意即该对象的内存在main()开始执行之前便已经分配好了，可以一直存在至程序结束。

内置类型的对象，如果定义在file scope之内，必定被初始化为0,。但如果它们被定义于local scope之内，那么除非程序员指定其初值，否则不会被初始化。

动态内存管理

不论是local scope或file scope，都是由系统自动管理。第三种储存期形式称为dynamic extent，其内存由程序的空间空间分配而来，有时也称heap memory。这种内存必须由程序员自行管理，分配系通过new表达式来完成，而其释放则通过delete表达式完成。

从heap分配而来的对象，被称为具有dynamic extent，因为它们是在运行时通过new表达式分配来的，因此可以持续存活，直到以delete表达式加以释放位置。删除是delete会自动检查指针是否为空。

2.4 使用局部静态对象

和局部非静态对象不同的是，局部静态对象所处的内存空间，即使在不同的函数调用过程中，依然持续存在。

2.5 声明inline函数

当一个函数里面调用了其他函数时，如果我们希望它在运行的时候能够直接展开而不是调用。那么我们可以将函数声明为inline函数，表示要求编译器在每个函数调用点上，将函数的内容展开。编译器可将该函数的调用操作改为以一份函数代码的副本代替。

将函数指定为inline，只是对编译器提出的一种要求。编译器是否执行这项请求，需视编译器而定。

2.6 提供重载函数

编译器会将调用者提供的实际参数拿来和每个重载函数的参数对比，找出其中最合适的。这也就是每个重载函数的参数列表必须和其他重载函数不同的原因。

2.9 设定头文件

这一节实际跑没有那么严格，当做代码规范去遵守。

头文件的扩展名，习惯上是.h。标准库例外，它们没有扩展名。

我们把函数声明放在头文件中，并在每个程序代码文件内包含(include)这些函数声明。

使用这种编程习惯，我们只需为函数维护一份声明即可。如果其参数列表或返回类型需要改变，也只需更改这份声明即可。

当我们在头文件中添加string、vector这些的时候，不要忘了写上using namespace std，否则会不能识别string。

函数的定义只能有一份，把函数声明放在头文件中，把函数定义放在程序代码文件中。

“只定义一份”的规则有个例外：inline函数的定义。为了能够扩展inline函数的内容，在每个调用点上，编译器都得取得其定义。这意味着我们必须将inline函数的定义放在头文件中，而不是把它放在不同的程序代码文件中。

3. 泛型编程风格

Standard Template Library(STL)主要有两种组件构成：一是容器(container)，包括vector、list、set、map等类；另一种组件是用以操作这些容器的所谓泛型算法(generic algorithm)，包括find()、sort()、replace()、merge()等。

vector和list是顺序性容器，我们主要进行迭代操作；map和set是关联容器，让我们快速查找容器中的元素值。

map是一对对的key/value组合。key用于查找，value用来表示我们要储存或取出的数据。举例map是电话号码，value是电话用户名。

set仅含有key。我们对它进行查询操作，为的是判断某值是否存在于其中。

泛型算法提供了许多可作用与容器类及数组类型上的操作。这些算法之所以被称为泛型，是因为它们和它们想要操作的元素类型无关。

泛型算法系通过function template技术实现泛型，诀窍就是不要直接在容器上进行操作，而是借由一对iterator(first和last)，标示我们要进行迭代的范围。那么iterator是？

3.1 指针的算术运算

当数组被传给函数，或是由函数中返回，仅有第一个元素的地址会被传递。

虽然array是以第一个元素的指针传入函数中，但仍然可以通过subscribe运算符访问array的每个元素。因为事实上所谓下标操作就是将array的起始地址加上索引值，产生出某个元素的地址，然后该地址再被提领以返回元素值。

如果array的第一个元素的地址为1000，那么array+2会产生什么地址呢？在指针运算中，会把”指针所指的类型”的大小考虑进去。假设array所储存的元素类型是int，那么array+2便是array的地址加上两个整数元素的大小。有些机器中int是4 byte，那么这个指针算术运算的答案便是1008。

在进行搜索时，数组的最后一个元素的下一个地址可以起到标兵的角色，用以指示我们的迭代操作何时完成，如果我们仅仅是将该地址拿来和其他元素的地址进行比较，而不对该地址进行读取或写入操作，那就没有问题。

3.2 了解Iterator泛型指针

现在让我们扩展find()的功能，令它也能支持标准库所提供的list类别。

list也是个容器，不同的是，list的元素以一组指针相互链接：前向指针指向下一个元素，后向指针指向上一个元素。因此，指针的操作并不适用于list，因为指针的算术运算必须首先假设所有元素都储存在连续空间里，然后才能根据当前的指针，加上元素大小后，指向下一个元素。

解决这个问题的办法是，在底层指针的行为之上提供一层抽象，取代程序原本的”指针直接操作”方式。我们把底层指针的处理统统放在次抽象层中，让用户无须直接面对指针操作。
