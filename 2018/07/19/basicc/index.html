2. 面向过程的编程风格

主要介绍独立函数，独立函数的好处在于1.取代重复编写相同的程序代码，可使程序更易懂；2.可以在不同的程序中使用；3.更容易分工协作。

2.1 如何编写函数

当函数返回值类型是void的时候，return语句用来提前结束函数的执行。

如果函数的返回类型不为void，那么它必须在每个可能的退出点上将值返回。函数中的每条return语句都被用来明确表示该处就是函数的退出点。如果函数体的最后一条语句不是return，那么最后一条语句之后便是该函数的隐式退出点。

另外如果没有在main()的末尾写下return语句，这一语句会被自动加上。

2.2 调用函数

本节将实现一个可对vector内的整数值加以排序的函数，通过这个例子，我们可以审视两种参数传递方式：传址及传值。

当我们调用一个函数时，会在内存中建立起一块特殊区域，称为”程序堆栈”，这块特殊区域提供了每个函数参数的储存空间。它也提供了函数所定义的每个对象的内存空间——我们将这些对象称为local object。一旦函数完成，这块内存就会被释放掉，或者说是从程序堆栈中被pop出来。

当我们将参数列表中的对象传入函数，默认情形下其值会被复制一份，成为参数的局部性定义。这种方式称为传值。

只要在类型名称和reference之间插入&符号，就是声明了一个reference。C++不允许我们改变reference所代表的对象，它们必须从一而终。

如果以传值方式传入display()之中，这意味着每当我想进行显示操作时，vector内的所有元素都会被复制。这并不会造成错误，但是如果直接传入vector的地址，速度会更快。

pointer参数和reference参数之间更重要的差异是，pointer可能不指向某个实际对象，当我们提领pointer时，一定要先确定其值并非0。

当函数返回一个对象的时候，不论以pointer或reference形式将对象返回，都不正确。因为在函数执行完毕时已经不存在了。如果以传值方式返回，便不会产生问题：因为返回的是对象的副本，在函数之外依然存在。

2.2.1 作用域及范围

为对象分配的内存，其存活时间称为范围(extent)。

对象在程序内的存活区域称为该对象的作用域(scope)。

局部性范围（local extent）：每次函数执行，都会为对象分配内存，每当函数结束便会加以释放，我们称此对象具有局部性范围。函数参数便具有局部性范围。

函数参数也是具有局部作用域。

对象如果在函数以外声明，具有所谓的file scope，从声明点至文件末尾都是可见的。

file scope内的对象也具备所谓的static extent，意即该对象的内存在main()开始执行之前便已经分配好了，可以一直存在至程序结束。

内置类型的对象，如果定义在file scope之内，必定被初始化为0,。但如果它们被定义于local scope之内，那么除非程序员指定其初值，否则不会被初始化。

动态内存管理

不论是local scope或file scope，都是由系统自动管理。第三种储存期形式称为dynamic extent，其内存由程序的空间空间分配而来，有时也称heap memory。这种内存必须由程序员自行管理，分配系通过new表达式来完成，而其释放则通过delete表达式完成。

从heap分配而来的对象，被称为具有dynamic extent，因为它们是在运行时通过new表达式分配来的，因此可以持续存活，直到以delete表达式加以释放位置。删除是delete会自动检查指针是否为空。
